## Decisions

- **Path B (Dylint + clippy_utils) chosen over Path A (extend syn matcher on stable).** Tradeoff accepted: captrack-pgo-lint pinned to nightly, ~2-monthly rebump; in return get semantic alias/Default/macro-expansion resolution via rustc internals. User explicitly accepted "одну последнюю стабильную версию" of clippy_utils.
- **captrack-pgo-lint is a SEPARATE workspace, not a member of captrack.** Otherwise its `rust-toolchain.toml` would force nightly on the whole repo. Root `Cargo.toml` gets `exclude = ["captrack-pgo-lint"]`.
- **Don't write a custom rustc-driver** — use Dylint framework + clippy_utils, much less boilerplate; standard distribution path via `cargo dylint`.
- **Old syn-based scan/apply gets DELETED in M5**, not kept as fallback. User's framing: "путь B правильный, другой неправильный". Profile loader, undo (manifest-based), report — survive; scan + apply replaced by Dylint pipeline.
- **`docs/CAPACITY-PGO-PLAN.md` kept as historical context** even though Phase 5 strategy changed mid-flight; will get a Path-B addendum in M5 rather than rewriting the original plan.
